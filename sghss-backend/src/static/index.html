Endpoints da API

GET
Health Check
/api/health
Verifica se a API está funcionando corretamente.
POST
Registro de Usuário
/api/auth/register
Cria um novo usuário no sistema (patient, professional, admin).
POST
Login
/api/auth/login
Autentica usuário e retorna token JWT para acesso aos endpoints protegidos.
GET
Listar Pacientes (Admin)
/api/patients
Lista todos os pacientes cadastrados. Requer autenticação como administrador.
POST
Criar Perfil de Paciente
/api/patients
Cria um perfil detalhado para usuários do tipo 'patient'. Requer autenticação.
GET
Meu Perfil de Paciente
/api/patients/me
Retorna o perfil do paciente logado com dados formatados.
PUT
Atualizar Meu Perfil
/api/patients/me
Atualiza dados do perfil do paciente logado com validações melhoradas.
GET
Informações do Usuário
/api/auth/me
Retorna informações detalhadas do usuário autenticado.
POST
Renovar Token
/api/auth/refresh
Renova o token de acesso usando o refresh token armazenado.
POST
Logout
/api/auth/logout
Registra o logout do usuário no sistema de auditoria.
